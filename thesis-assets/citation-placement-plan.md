## 当前暂不强行落位的位置

### 第二章原理性技术描述

第二章中以下内容属于“技术原理/官方能力/框架机制”表述，当前不建议直接用现有研究文献硬挂：

- Transformer 架构基础原理
- OpenAI 协议标准化带来的接口生态意义
- SSE 与 `EventSource` / `fetch + ReadableStream` 的工程差异
- Vue 3 Composition API 的框架级设计特点
- Spring Boot 3 异步线程池机制
- Apache PDFBox 3.0 文本提取能力

原因很直接：这些内容更适合用官方文档、标准文档或框架资料作为依据，而不是拿研究论文去替代官方技术来源。

因此，第二章建议分成两类引用：

1. **原理与框架能力**：后续用官方文档补。
2. **与你系统设计直接相关的研究约束**：可用当前文献包补。

其中，当前文献包已经足够支持的仅包括：

- 多轮交互中的上下文保持、连贯性与鲁棒性挑战，可用 `[4][5]`
- 面向多轮系统的上下文工程优化方向，可用 `[6]`
- 简历语义匹配研究背景，可用 `[2][8]`
