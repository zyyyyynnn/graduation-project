# 第三章与第五章条件性清稿策略

## 目标

本文件用于明确第三章、第五章为什么暂时不能像第一章、第二章、第六章那样直接做低风险清稿，以及它们各自进入正式清稿前所必须满足的条件。

## 第三章：系统分析与设计

### 当前不能直接彻底清稿的原因

第三章当前最大的阻塞不是文字本身，而是**图表引用尚未最终落位**。章节末尾现有协作说明虽然不适合进入终稿，但它实际承担了一个提醒作用：当前正文还没有写成正式的“如图 3-1 所示”“如图 3-2 所示”“如图 3-3 所示”。

如果现在直接把相关提示删掉，而你后续又没有及时补图号、图名与图表顺序，就容易出现：

1. 第三章正文缺少对架构图、用例图、E-R 图的正式引用。
2. 图表编号和正文描述脱节。
3. 答辩 PPT 与论文图号不一致。

### 第三章进入正式清稿的前置条件

至少满足以下 3 项：

1. 明确图号：
   - 系统架构图编号
   - 用例图编号
   - E-R 图编号

2. 明确图名：
   - 图 3-1 名称
   - 图 3-2 名称
   - 图 3-3 名称

3. 明确表格编号策略（如数据库设计表是否在本章出现）

### 第三章清稿时要做的动作

在条件满足后，我会一次性完成：

1. 删除标题中的“（草稿）”
2. 删除章节末尾 `## 备注`
3. 将“后续可补图号”的协作说明改写为正式正文引用
4. 统一第三章中的图号、图名与表号口径

## 第五章：系统测试

### 当前不能直接彻底清稿的原因

第五章当前不是“措辞还不够正式”，而是**证据还没有回填**。测试环境、功能测试结果、TTFB、PDF 解析耗时、通过情况都仍是空位。如果现在仅做机械清稿，会产生两个风险：

1. 把尚未完成的测试章节伪装成已完成状态。
2. 提前删除对“待回填”的提醒，导致后续漏填真实数据。

### 第五章进入正式清稿的前置条件

至少满足以下 4 项：

1. 真实测试环境参数
   - Windows 版本
   - JDK 版本
   - MySQL 版本
   - 浏览器版本
   - 网络环境

2. 功能测试结果
   - `TC-01` 到 `TC-09` 的实际结果
   - 是否通过

3. 性能测试结果
   - SSE TTFB
   - PDF 解析耗时
   - 如有，测试次数或区间

4. 测试截图或证据
   - DevTools Network 截图
   - 页面截图
   - 如有，数据库或接口响应截图

### 第五章清稿时要做的动作

在条件满足后，我会一次性完成：

1. 删除标题中的“（草稿）”
2. 删除章节末尾 `## 备注`
3. 将“待回填”全部替换为真实测试结果
4. 将“当前阶段尚未嵌入真实环境参数”等过渡语句收敛成正式测试表述
5. 同步回调第六章摘要与总结中的测试表述

## 当前策略结论

### 现在就能做的

1. 建立清稿前置条件清单
2. 固化第三章和第五章的补料边界
3. 提前准备好图号登记表与测试数据模板

### 现在不该做的

1. 不应提前删除第三章图表占位说明而不补正式图号
2. 不应提前删除第五章“待回填”而不补真实测试结果
3. 不应把第五章现状误写成正式测试结论

## 后续执行顺序

建议按以下顺序推进：

1. 你补第三章图号与图名
2. 我完成第三章正式清稿
3. 你补第五章真实测试数据与截图
4. 我完成第五章正式清稿并回调第六章测试相关表述
