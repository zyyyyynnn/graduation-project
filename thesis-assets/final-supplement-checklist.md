# 终稿前最终补料清单

## 说明

这份清单只保留**终稿收束前最低必要**的真实补料项。你不需要再按章节自己拆分，只需要按资料类型补给我，我会负责落回对应章节。

## A. 图表补料

### 必补

1. 第三章图号与图名
   - 系统架构图
   - 用例图
   - E-R 图

2. 若第五章有正式测试表编号，也请一并确认

### 推荐附带

1. 图表文件名
2. 图表最终版本截图或 PNG
3. 图表编号登记表

## B. 测试数据补料

### 必补

1. 测试环境
   - Windows 版本
   - JDK 版本
   - MySQL 版本
   - 浏览器版本
   - 网络环境

2. 性能数据
   - SSE TTFB
   - PDF 解析耗时

3. 功能测试结果
   - `TC-01` 到 `TC-09` 的实际结果
   - 是否通过

### 推荐附带

1. DevTools Network 截图
2. 页面测试截图
3. 接口返回截图

## C. 实现证据补料

### 必补

如果你希望第四章和答辩部分更扎实，建议至少补：

1. 3 组核心代码片段
   - SSE 推送
   - PDF 解析
   - 上下文截断

2. 2 个真实 Bug 记录
   - 现象
   - 排查过程
   - 解决方案
   - 教训

### 推荐附带

1. 数据库 `interview_message` 表截图
2. 流式过程截图
3. 报告页截图

## D. 答辩补料

### 必补

1. 你真实开发中遇到的关键难点
2. 你希望重点展示的系统亮点
3. 如有，老师可能追问的薄弱点

### 推荐附带

1. 演示视频或演示顺序
2. PPT 初稿或页映射

## 建议的最省事提交方式

你后面只需要按下面方式分 2 次补料即可：

### 第一次补料

- 图表补料
- 测试数据补料

### 第二次补料

- 实现证据补料
- 答辩补料

## 我收到后会做什么

### 收到第一次补料后

我会完成：

1. 第三章正式清稿
2. 第五章正式清稿
3. 第六章测试相关表述回调

### 收到第二次补料后

我会完成：

1. 第四章增强版收束
2. 答辩材料整理
3. 终稿前最后总审
