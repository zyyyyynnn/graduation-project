# 真实信息收集记录（2026-04-18）

## 1. 本轮代码审查结论

本轮审查范围覆盖前端最简单页、后端认证、简历、岗位、面试会话、SSE 流式对话与报告生成主链路。

本轮发现并修复的前端接口契约问题：

1. 面试回答请求字段原为 `answer`，后端 DTO 实际字段为 `content`，已改为 `content`。
2. 简历上传返回字段原按 `id/fileName` 使用，后端实际返回 `resumeId/skills/projects`，已改为使用 `resumeId`，文件名使用本地上传文件名展示。
3. 前端默认登录密码原为 `demo123456`，数据库初始化用户 `demo` 的真实密码校验为 `123456`，已改为 `123456`。
4. 本地已有 token 时，前端现在会自动恢复并加载简历列表与岗位列表。

当前仍保留的边界：

1. 未执行真实 DeepSeek 成功调用，因此 PDF 成功结构化解析、有效 SSE 模型回答、报告生成质量仍需后续真实链路补测。
2. 前端采用最简黑白单页，满足演示主流程，不再保留复杂 UI、路由与状态管理。
3. 自动化单元测试目录当前不存在，本轮验证以构建、启动和接口实测为主。
