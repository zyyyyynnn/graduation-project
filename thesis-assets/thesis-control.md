## 使用方式

- 日常只维护“当前阶段、当前卡点、下一步缺什么”这几个最小信息。
- 论文正文、文献、测试、图表、答辩材料的详细文件索引见 `thesis-assets/README.md`。
- 所有产出必须以真实材料和已确认信息为准；不得补造学校模板、真实公网 LLM 性能数据或不存在的系统能力。

## 后续优先级

### P0：终稿前必须处理

1. 人工填写或确认封面、诚信责任书中的指导教师、签名、日期等字段。
2. 优先检查 `current/毕业论文正式版（润色回填）.docx`，在 Word 中更新自动目录或手工校正目录页码。
3. 逐页检查页眉、页脚、页码、图题、表题和参考文献格式。
4. 确认参考文献中普通期刊、预印本、官方文档是否允许进入最终稿。

### P1：答辩前处理

1. 人工重新提供或从历史提交恢复 PPT 模板后，生成正式答辩 PPT。
2. 根据 `defense/defense-slide-map.md` 控制 10-12 页主线内容。
3. 本机完整跑通 Demo Twin，并准备离线截图兜底。
4. 根据导师关注点补充 8-10 个高频追问口径。

### P2：可选增强

1. 如导师要求真实性能对比，再使用真实 API Key、真实网络和真实模型服务单独采集。
2. 如需要更强测试证据，可补 DevTools Network 截图和数据库表截图。
3. 如学校要求查重前降重，可另做终稿语言核查。

## 阶段断点规则

- 论文终稿人工字段、目录域和格式核查完成后：提交 DOCX/PDF 供人工审查。
- 答辩 PPT 完成后：提交 PPT、讲稿和演示路线供人工审查。
- 真实公网 LLM 性能测试仅在导师明确要求时进行，不作为当前草稿阻塞项。
