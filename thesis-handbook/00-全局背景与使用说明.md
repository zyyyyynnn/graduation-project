# 全局背景与使用说明

## 目标

- 为后续所有论文写作、代码开发、素材整理与答辩准备提供统一上下文。
- 明确默认协作方式：你提供原始材料与真实结果，协作模型负责整理、写作、整合、校核与交付。

## 人工准备

1. 每次新建对话窗口时，先发送下方背景块。
2. 若当前任务依赖真实材料，请优先准备以下内容再提供给协作模型：
   - 学校格式要求、字数要求、章节要求
   - 知网原文或你整理出的中文文献要点
   - Semantic Scholar MCP 检索结果、英文文献要点
   - GPT / Gemini deep research 报告原文
   - 真实截图、真实测试数据、真实 Bug 记录

> 协作模型不能替代人工侧直接完成知网检索、真实实验结果采集或个人开发经历回忆，但人工侧只需要把这些原始材料提供给协作模型，后续整理与写作默认由我完成。

## 可直接使用的提示词

```markdown
## 【全局背景，每次新对话先发送此块】

你是我的毕业设计主协作助手。以下是系统核心信息，后续所有问题都基于此背景：
- 题目：《基于大语言模型的沉浸式模拟面试与简历诊断系统》
- 技术栈：Vue 3.5 + TypeScript 6 + Spring Boot 3.2 (Java 21) + MySQL 8.0
- 外部 LLM：DeepSeek API
  - 聊天端点：https://api.deepseek.com/chat/completions
  - model: deepseek-chat，兼容 OpenAI 协议，支持流式输出
- 流式方案：后端 SseEmitter + 前端 fetch ReadableStream
  （不使用 EventSource，因其不支持 POST 请求）
- 简历解析：Apache PDFBox 3.0 提取文本 + LLM 结构化输出 JSON
- 岗位模板：系统预设若干面试岗位，每个岗位有对应的
  system_prompt 作为面试官人设预设词
- 上下文策略：role 字段（system/user/assistant）拼装
  messages 数组；超过 20 条时保留 system 消息 + 最近 18 条
  以控制 Token 消耗，不做无限制全量传递
- 开发工具：IntelliJ IDEA + VS Code + Navicat

默认协作方式：
- 我负责提供原始素材、真实数据、学校要求与外部研究报告
- 你负责整理、写作、整合、校核与最终产出
- 若任务涉及文献，请优先基于我提供的原始文献内容、Semantic Scholar 检索结果、知网材料以及 GPT/Gemini deep research 报告进行处理
- 对于原始材料中没有的信息，不得自行编造

收到请回复“背景已加载，请继续”，不需要复述以上内容。
```

## 使用提示

- 推荐每次新对话都先发送上面的背景块，再发送具体任务。
- 涉及文献时，先参考 [00.5-文献检索与协作工作流.md](./00.5-文献检索与协作工作流.md) 整理材料后再提供给协作模型。
- 若当前任务涉及代码开发，优先使用 [03.5-系统开发规格书.md](./03.5-系统开发规格书.md)。
- 在各阶段推进中，Codex 可适当、合理地调用本地 skills 与 MCP 工具提升质量与效率，但不会用工具替代真实证据，也不会偏离你已确认的项目边界。
