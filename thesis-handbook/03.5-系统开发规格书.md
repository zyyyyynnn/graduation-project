## 3. 接口规格

### 3.0 全局约定

- 统一响应体 `Result<T>`
- JWT 认证，排除 `/api/auth/**` 和 `/api/health`
- 跨域允许 `localhost:5173`

---

### 一期：Provider 抽象层 + 用户 API Key

#### GET /api/llm/providers
返回所有 `enabled=1` 的 Provider 列表（含可选模型）。无需认证。

#### PUT /api/user/llm-config
请求体：`{ "providerKey": "openai", "model": "gpt-4o", "apiKey": "sk-..." }`

硬约束：
- `apiKey` 非空时，AES-256-GCM 加密后存库，**明文不落库**
- `apiKey` 为空字符串时，清空用户自定义 Key，回退到系统默认
- 加密 Key（AES secret）只能来自 `application-local.yml`，禁止硬编码

#### GET /api/user/llm-config
返回当前用户的 Provider、模型配置，**apiKey 字段只返回脱敏后的末 4 位**，禁止返回完整密文。

---

### 二期：补齐基础功能 + 面试体验

#### DELETE /api/resume/{resumeId}
硬约束：
- 校验 resumeId 属于当前用户
- 若存在关联的 `interview_session`（任意状态），禁止删除，返回 400

#### GET /api/interview/{sessionId}/messages
返回该会话的完整对话记录（含 system 消息）。
硬约束：校验 sessionId 属于当前用户。

#### POST /api/interview/{sessionId}/chat（变更）
在原有基础上新增：
- 首条用户消息（`seq_num=1`）发送前，后端检查是否需要 AI 主动发第一问（由 `?autoStart=true` 参数触发）
- 硬约束：`autoStart=true` 时，`content` 字段可为空字符串，后端注入引导词后调用 LLM

#### POST /api/interview/{sessionId}/stage
请求体：`{ "stageName": "technical" }`
切换面试阶段，写入 `interview_stage` 表（`stage_name` 字段），同时向 LLM 上下文注入阶段转换提示词。
硬约束：阶段只能向后推进（warmup → technical → deep_dive → closing），不允许回退。

#### PUT /api/user/profile
请求体：`{ "email": "new@example.com", "oldPassword": "...", "newPassword": "..." }`
硬约束：
- 修改密码必须提供 `oldPassword` 验证
- `newPassword` 与 `oldPassword` 不能相同
- email 和 password 可独立修改，不要求同时提供

---

### 三期：数据分析

#### GET /api/analytics/radar
返回当前用户近 N 场面试（默认 10）的三维平均分，供前端渲染雷达图。
响应结构：`{ "technical": 7.2, "expression": 6.8, "logic": 7.5, "sessionCount": 10 }`

#### GET /api/analytics/trend
返回当前用户按时间排序的历史评分列表，供前端渲染折线图。
响应结构：`[{ "sessionId": 1, "createdAt": "...", "technical": 7, "expression": 6, "logic": 8 }]`

#### GET /api/analytics/weaknesses
返回当前用户的薄弱点列表，按 `category` 分组，出现频次降序。

#### POST /api/interview/{sessionId}/finish（变更）
在原有生成报告基础上，额外完成：
1. 报告 prompt 必须要求 LLM 输出**三个维度**评分：技术能力、表达清晰度、逻辑思维（各 1-10 分），用于后续结构化提取写入 `score_history`
2. 从报告文本中提取三维评分，写入 `score_history`
3. 调用 LLM 提取薄弱知识点，写入 `user_weakness`
4. 第 2、3 步失败不影响主流程（try-catch 降级，日志记录）

## 4. LLM 抽象层硬约束

- 定义 `LlmProvider` 接口，方法：`chat()`、`streamChat()`
- 每个 Provider 实现类（`DeepSeekProvider`、`OpenAiProvider` 等）注册为 Spring Bean
- `LlmUtil` 重构为 `LlmRouter`，根据 `UserContext` 中的用户配置动态路由到对应 Provider
- **优先级**：用户自定义 Key > 系统默认 Key，两者均未配置时返回 400
- Provider 实现类禁止感知用户信息，路由逻辑只在 `LlmRouter` 层处理
- 会话创建时（`/start`）将实际使用的 provider + model 快照写入 `interview_session`，后续该会话的所有请求使用快照值，不受用户中途修改配置影响

## 5. 安全约束

| 约束 | 实现要求 |
|---|---|
| 用户 API Key | AES-256-GCM 加密存库，加密 secret 只来自配置文件 |
| API Key 返回 | 接口只返回末 4 位脱敏，禁止返回完整密文或明文 |
| DeepSeek 系统 Key | `application-local.yml`，.gitignore，`@Value` 注入 |
| JWT 密钥 | 最小 32 字节，构造函数校验，不满足则阻止启动 |
| 用户资源隔离 | 所有 resumeId、sessionId 均校验属于当前用户 |
| ThreadLocal 清理 | 请求完成 + SSE 异步线程 finally 块均调用 `UserContext.remove()` |
| SSE 异步线程 | 入口处重新注入 userId，finally 块清理 |
| 密码存储 | BCrypt 单向加密，修改密码必须验证旧密码 |

## 6. 错误边界约束

### 后端

| 场景 | 处理 |
|---|---|
| 用户 API Key 解密失败 | 降级使用系统默认 Key，日志记录，不暴露加密细节给前端 |
| Provider 不可达 | 400，提示"模型服务暂不可用，请稍后重试或切换 Provider" |
| 简历有关联会话时删除 | 400，提示"该简历已被面试使用，无法删除" |
| 面试阶段回退请求 | 400，提示"面试阶段不可回退" |
| 评分提取失败（三期） | 降级跳过，不影响报告生成主流程 |
| 薄弱点提取失败（三期） | 同上 |

### 前端

| 场景 | 处理 |
|---|---|
| Provider 切换后当前会话不受影响 | 路由到快照 provider，前端无需处理 |
| 雷达图数据为空（无历史） | 展示空态引导文案，不渲染图表 |
| 模型对比实验页 LLM 调用失败 | 单个 Provider 失败不阻塞其他，独立展示错误状态 |

## 7. 各期交付范围

### 一期
- `llm_provider_config` 表 + 初始数据（DeepSeek、OpenAI）
- `user` 表新增 LLM 配置字段
- `LlmProvider` 接口 + `DeepSeekProvider` / `OpenAiProvider` 实现
- `LlmRouter` 替换原 `LlmUtil`
- 用户 API Key 加密存取
- `/api/llm/providers`、`PUT /api/user/llm-config`、`GET /api/user/llm-config`
- 前端：设置页 LLM 配置面板

### 二期
- `interview_stage` 表
- 简历删除接口
- 对话回放接口
- AI 主动发第一问（`autoStart` 参数）
- 面试阶段切换接口 + 阶段提示词注入

- `score_history` 表、`user_weakness` 表
- `/finish` 接口扩展（三维评分写入、薄弱点提取）
- 数据分析三个接口（radar、trend、weaknesses）
- 前端：数据看板页（雷达图 + 折线图 + 薄弱点列表）

## 8. UI 设计规范

前端所有页面严格遵循 `DESIGN-SPEC.md`，以下为调用约定：

### 字体引入
在 `index.html` `<head>` 中引入，不得在组件内单独引入：
```html
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=Lora:wght@400;500&family=Inter:wght@400;500&family=JetBrains+Mono:wght@400&display=swap" rel="stylesheet">
```

### CSS 变量
在 `src/styles/index.css` 的 `:root` 中统一声明全部设计 Token，组件内通过变量引用，禁止硬编码颜色值。

### Element Plus 主题覆盖
在 `src/styles/index.css` 中覆盖 Element Plus CSS 变量，使其与设计 Token 对齐：
```css
--el-color-primary: #c96442;
--el-border-radius-base: 8px;
--el-font-size-base: 16px;
--el-bg-color: #faf9f5;
```

### 装饰 SVG 使用约束
- 所有装饰 SVG 必须带 `aria-hidden="true"` 和 `pointer-events: none`
- 缝线气泡只用于面试对话页消息气泡，其他页面禁止使用
- 线条填充字形只用于大标题装饰（≥80px），禁止用于功能性文字
- 线框几何装饰只用于背景层，z-index 低于内容层
