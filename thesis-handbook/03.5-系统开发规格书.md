## 数据库 DDL（MySQL 8.0，utf8mb4，InnoDB）

生成以下 5 张表的完整建表 DDL，含索引与 COMMENT：

1. user：id(PK)、username(UNIQUE)、password(BCrypt加密)、
   email、created_at
2. resume：id(PK)、user_id(FK→user)、file_name、
   parsed_skills(TEXT)、parsed_projects(TEXT)、
   raw_text(MEDIUMTEXT，存PDFBox提取原文)、created_at
3. position_template：id(PK)、name(岗位名)、
   system_prompt(TEXT，面试官人设预设词)
4. interview_session：id(PK)、user_id(FK→user)、
   resume_id(FK→resume)、position_id(FK→position_template)、
   target_position(VARCHAR，冗余存岗位名便于展示)、
   status(ENUM('ongoing','finished')，英文值防编码问题)、
   summary_report(TEXT，AI评估报告Markdown内容)、created_at
5. interview_message：id(PK)、session_id(FK→interview_session)、
   role(ENUM('system','user','assistant'))、
   content(TEXT)、
   seq_num(INT，消息序号，用于有序拼装上下文，后端自动维护)、
   created_at

## 后端实现要求

### 生成顺序（严格遵守，后续模块依赖前序基础设施）
基础设施 → 认证模块 → 简历解析 → 面试会话 → 流式对话 → 报告生成

### 基础设施
- Result<T>：统一返回体 { code, message, data }
- BusinessException：自定义业务异常，含 code 与 message
- 全局异常处理：@RestControllerAdvice，处理
  BusinessException 与 MethodArgumentNotValidException
- JWT 认证：
  - jjwt 库，token 有效期 7 天
  - JwtInterceptor 从 Authorization Header 提取并验证 token
  - UserContext：ThreadLocal<Long> 存当前用户 id，
    拦截器后置必须调用 UserContext.remove() 防内存泄漏
  - WebMvcConfigurer 注册拦截器，排除 /api/auth/**
- 跨域：全局 CorsFilter，允许 localhost:5173
- 线程池：ThreadPoolTaskExecutor，核心线程 5，最大 20，
  队列 100，命名前缀 sse-pool-，用于 SSE 异步推送

### ORM 策略
优先使用 MyBatis-Plus 内置方法（selectById、selectList +
QueryWrapper、save、updateById）。
仅当涉及多表 JOIN 等复杂 SQL 时才使用 XML 映射文件。
禁止在简单 CRUD 场景滥用 XML。

### 模块 1：用户认证
- POST /api/auth/register：BCrypt 加密密码，用户名唯一性校验
- POST /api/auth/login：验证通过后返回 JWT token

### 模块 2：简历解析
- POST /api/resume/upload：
  - 文件校验：仅接受 PDF，大小 ≤ 10MB
  - PDFBox 提取文本：使用 PDFTextStripper，
    设置 setSortByPosition(true) 改善中文排版乱序
  - 组装提示词调用 DeepSeek API（同步，非流式），
    要求返回严格 JSON：
    { "skills": ["技能1","技能2"], "projects": [
      {"name":"项目名","description":"项目描述"} ] }
  - Jackson 反序列化，Jackson 解析失败时抛出 BusinessException
    而非直接崩溃
  - 入库 resume 表，返回 resumeId 与解析结果
- GET /api/resume/list：查询当前用户简历列表

### 模块 3：面试会话
- GET /api/position/list：返回全部岗位模板列表
- POST /api/interview/start：
  - 创建 interview_session 记录（status='ongoing'）
  - 根据 position_id 查询 system_prompt
  - 插入第一条 role='system' 的 interview_message，
    seq_num=0，内容为 system_prompt
  - 返回 sessionId
- GET /api/interview/sessions：查询当前用户历史面试列表

### 模块 4：流式对话（核心模块）
- POST /api/interview/{sessionId}/chat
  请求体：{ "content": "用户回答文本" }

  处理逻辑（必须在异步线程池中执行，严禁阻塞主线程）：
  1. 校验 sessionId 属于当前用户，status='ongoing'
  2. 将用户消息插入 interview_message：
     - seq_num = 查询当前 session 最大 seq_num + 1
     - role='user'，content=用户回答文本
     （seq_num 由后端自动维护，不接受前端传入）
  3. 按 seq_num 升序查询该 session 全部消息，
     构建 messages 数组：
     - 若总条数 > 20，保留第 1 条（system）+ 最后 18 条
     - 其余完整传入
  4. 调用 DeepSeek 流式 API：
     - stream: true
     - 逐行读取响应，解析 data: 前缀
     - 内容为 [DONE] 时调用 emitter.complete() 结束
     - 否则提取 choices[0].delta.content，
       通过 SseEmitter.send() 推送给前端
  5. 流式结束后将完整 AI 回复拼接，
     插入 interview_message（role='assistant'，seq_num 继续递增）

  SseEmitter 配置：
  - 超时：120000ms（120秒）
  - onTimeout：发送错误事件后 complete()
  - onError：记录日志后 complete()
  - onCompletion：释放资源（无需额外操作，Spring 自动处理）

### 模块 5：评估报告生成
- POST /api/interview/{sessionId}/finish
  1. 校验 session 属于当前用户且 status='ongoing'
  2. 查询全部对话记录，组装评估提示词，要求 LLM 以
     Markdown 格式输出：
     - 技术能力匹配度（1-10分 + 说明）
     - 表达清晰度（1-10分 + 说明）
     - 核心优势总结
     - 改进建议（3条）
  3. 同步调用 DeepSeek（非流式），结果存入
     interview_session.summary_report
  4. 更新 session status='finished'
  5. 返回报告内容

## 前端实现要求

### 基础设施
- Axios 封装：
  - 请求拦截器：自动附加 Authorization: Bearer {token}
  - 响应拦截器：HTTP 非 200 状态码统一 ElMessage.error() 提示；
    401 状态清除 token 并跳转 /login
- Pinia userStore：存储 token 与用户信息，
  使用 pinia-plugin-persistedstate 持久化到 localStorage
- Vue Router：路由守卫，未登录用户重定向到 /login
- 环境变量：.env.development 中配置
  VITE_API_BASE_URL=http://localhost:8080，
  所有接口地址通过此变量注入，禁止硬编码

### 页面 1：LoginView.vue
- Tab 切换登录/注册表单，表单校验
- 登录成功存 token，跳转简历上传页

### 页面 2：ResumeUploadView.vue
- el-upload 拖拽上传，限制 PDF + 10MB，
  超出限制给出明确提示
- 上传中显示全屏 Loading："AI 正在解析您的简历..."
- 解析成功：技能展示为 el-tag，项目展示为卡片列表
- **错误边界**：解析失败或超时必须清空 Loading 状态，
  展示友好重试提示，严禁永久 Loading
- 底部"开始面试"按钮，弹出岗位选择对话框
  （从 /api/position/list 拉取，el-select 展示）

### 页面 3：InterviewView.vue（核心页面）
- 聊天气泡布局：用户消息右侧蓝色，面试官左侧灰色
- 使用 useInterviewStream.ts 发起流式请求：
  - 流式接收时追加字符到 ref，实现打字机效果
  - 四种状态管理：idle / connecting / streaming / done / error
  - **中文 TCP 粘包处理**：buffer 累积不完整的 UTF-8 字节，
    按 \n\n 分割完整的 SSE 事件，避免截断中文字符
  - **错误边界**：SSE 断流或异常时显示
    "连接已断开，请重试"，并重置发送按钮为可用状态，
    严禁按钮永久禁用（死锁）
- 输入框：回车发送，流式进行中禁用输入与发送按钮
- 顶部展示目标岗位名称；右上角"结束面试"按钮
  调用 /finish 接口后跳转报告页

### 页面 4：ReportView.vue
- 调用 /finish 接口后展示评估报告
- 使用 markdown-it 渲染 Markdown 格式报告内容
- 技术匹配度与表达清晰度用 el-progress 进度条展示
- 提供"再次面试"与"返回首页"按钮
