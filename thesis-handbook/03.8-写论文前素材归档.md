# 写论文前素材归档

## 目标

- 明确你只负责采集真实素材，我负责告诉你需要哪些素材以及后续如何落到论文中。
- 为第四章实现、第五章测试和答辩展示准备可靠的原始证据。

## 人工准备

### 先完成的归档动作

1. 建立 `thesis-assets/` 文件夹，包含以下子目录：
   - `screenshots/`
   - `code-snippets/`
   - `diagrams/`
   - `test-data/`
2. 采集真实素材并发给我或按目录落盘：
   - 简历上传 + 解析结果页截图
   - 流式打字机进行中页面截图
   - 评估报告页截图
   - 用户历史记录页截图
   - Postman 或浏览器 Network 面板的 SSE 截图
   - 数据库 `interview_message` 表实际数据截图
   - SSE 推送、PDF 解析、上下文拼装三类核心代码片段
   - Mermaid 图导出的 PNG
   - DevTools Network 面板的 TTFB 实测截图
3. 如素材已接近定稿，开始填写最终证据锁版记录，标记每份截图、测试数据和论文/PPT 的同步情况。
4. 如图表较多，开始填写图表编号登记表。
5. 手写 `dev-notes.md`，记录至少 2 个真实 Bug。

> 真实截图、真实代码片段、真实测试结果和真实 Bug 记录必须由你提供；我会基于这些素材告诉你还能补什么，以及每份素材在论文中如何使用。

## 可直接使用的提示词

### 素材盘点与缺口检查

```markdown
我现在进入素材归档阶段。以下是我已经采集到的真实素材，请你帮我做一次盘点，告诉我：
1. 还缺哪些关键素材；
2. 每类素材适合放到论文或答辩的哪个位置；
3. 文件命名和归档方式是否需要调整；
4. 哪些素材最值得优先补齐。

当前已有素材：
[在此粘贴截图清单、代码片段清单、测试数据清单、图表清单、Bug 记录摘要]
```

### `dev-notes.md` 记录模板

```markdown
# dev-notes.md 记录模板

## Bug 1

**问题**：流式接口偶发性在收到约一半内容后停止推送

**现象**：Network 面板显示 SSE 连接正常建立，data 事件在
收到部分内容后停止触发，没有 error 事件，前端打字机渲染中断

**排查过程**：在 Service 层加入日志后发现，DeepSeek 流式
响应的最后一个 chunk 内容是 [DONE]，代码未做判断直接尝试
解析 choices[0].delta.content，该字段为 null，
导致 NullPointerException，异步线程崩溃退出，
SseEmitter 未调用 complete()

**解决方案**：在解析每个 chunk 前判断内容是否为 [DONE]，
是则调用 emitter.complete() 正常关闭连接

**教训**：OpenAI 协议的流式响应以 data: [DONE] 作为终止标志，
这是必须处理的边界条件，不处理会导致前端连接异常挂起

---

**解决方案**：
**教训**：
```

## 使用提示

- 不要等写论文时再回忆 Bug，开发完成后立刻记录最准确。
- 你只需把已有素材和缺失项发给我，我会告诉你还差什么、优先补什么。
- 若需要直接落盘的素材目录模板，可复制 [assets-template](./assets-template/) 下的结构。
- 若素材已进入论文或 PPT，建议同步维护 [assets-template/final-evidence-lock-template.md](./assets-template/final-evidence-lock-template.md)。
- 若你开始给图表编号，建议同步维护 [assets-template/figure-table-register-template.md](./assets-template/figure-table-register-template.md)。
