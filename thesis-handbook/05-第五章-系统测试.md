# 第五章：系统测试

## 目标

- 在你提供真实测试结果与真实环境信息后，由我完成第五章【系统测试】的组织与写作。
- 兼顾测试用例表、性能描述与结论表达。

## 人工准备

1. 提供真实测试环境信息：
   - 操作系统
   - JDK 版本
   - MySQL 版本
   - 浏览器版本
2. 提供真实测试结果：
   - SSE 首字节响应时间（TTFB）
   - PDF 解析耗时
   - 功能测试通过 / 失败情况
3. 对性能数据补充最基本的测量条件：
   - 每项数据的测量次数或样本量
   - 测量方式
   - 当时的网络环境或说明
4. 补充关键边界与安全相关测试结果：
   - 未登录访问是否被拦截
   - 是否能访问他人的 session 或 resume
   - 空内容 / 恶意输入的处理结果
   - SSE 断流后的页面状态与恢复情况
5. 若某些测试失败或表现不稳定，请把真实现象一起发给我。

> 真实测试数据必须由你提供；测试章节的文字组织、表格生成与结论表达默认由我完成。

## 使用提示

- 不需要你自己先写测试表，我可以基于你给出的真实结果直接生成。
- 若你已经有 DevTools 截图或接口测试截图，也可以一起发给我，用于补强描述。
- 性能测试部分重点是“真实测到了什么”，不是复杂压测。
- 如果你还没做权限或越权测试，建议先补这类结果；老师追问时这部分很常见。
