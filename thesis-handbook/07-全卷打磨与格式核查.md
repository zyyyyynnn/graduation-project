# 全卷打磨与格式核查

## 目标

- 由我主导完成学术润色、查重降重、全文逻辑检查、引用一致性核查与排版格式核查。
- 明确区分不同任务，避免把多个目标混进同一次处理。
- 在不改变核心事实、数据和论证结论的前提下，提升论文表达质量与整体一致性。

## 人工准备

1. 逐章提供原文，不要一次性塞入整篇全文。
2. 提供学校或目标期刊的格式要求。
3. 若涉及查重、引用或参考文献问题，提供原始条目、红标段落或对应章节内容。
4. 若文末参考文献尚未整理规范，先参考 [07.5-参考文献整理.md](./07.5-参考文献整理.md) 完成条目整理。

> 润色、降重、逻辑检查、引用核查与排版终审必须分开执行。你只需要把对应材料逐项发给我，我来分别处理。

## 可直接使用的提示词

### 7A：深度学术润色

```markdown
我现在要做论文润色。以下是某一章节或某一段的原文，请你只基于原文进行学术化润色，不要编造任何数据、文献或结论。

原文：
[在此粘贴]

要求：
1. 保持原意不变
2. 优化逻辑与语言
3. 减少机械化和模板化表达
4. 若原文论据不足，只指出缺口，不自行补造
```

### 7B：查重降重

```markdown
我现在要处理查重标红段落。以下是原文，请你先判断它属于“客观原理描述”还是“可重组表达段落”，再决定改写策略。

原文：
[在此粘贴]

要求：
1. 不改变技术事实
2. 不替换专业术语
3. 说明主要采用了哪些降重策略
4. 若不建议继续强行降重，请明确指出原因
```

### 7C：全文逻辑检查

```markdown
我现在要做全文逻辑检查。以下是某一章原文，以及上一章的问题摘要（如有），请你只指出逻辑问题、术语不统一、章节断裂或图表引用缺失，不直接改写原文。

章节原文：
[在此粘贴]

上一章问题摘要（如有）：
[在此粘贴]
```

### 7D：文内引用与文末参考文献核查

```markdown
我现在要检查正文引用与文末参考文献是否对应。以下是正文片段和文末参考文献列表，请你只检查对应关系、编号顺序、前后一致性与格式规范性，不新增任何文献。

正文片段：
[在此粘贴]

参考文献列表：
[在此粘贴]
```

```markdown

[在此粘贴]

[在此填写]

[在此填写]
- 系统截图 / 图表：
[在此填写]
- 演示视频：
[在此填写]
- 代码或项目压缩包：
[在此填写]
- 其他学校要求材料：
[在此填写]
- 最终证据锁版记录（如有）：
[在此填写]

1. 列出缺失项

3. 给出最终提交前的核查顺序
4. 生成一份可逐项勾选的提交清单
```

## 使用提示

- 每次只做一种任务，效果最好。
- 如果你不确定应该先做哪一步，通常顺序是：参考文献整理 → 润色 → 降重 → 逻辑检查 → 引用核查 → 格式终审 → 最终提交包核查。
- 文献相关内容若涉及原始证据，优先参考 [00.5-文献检索与协作工作流.md](./00.5-文献检索与协作工作流.md)。
- 若你已经进入提交前阶段，可把 [assets-template/final-evidence-lock-template.md](./assets-template/final-evidence-lock-template.md)、[assets-template/figure-table-register-template.md](./assets-template/figure-table-register-template.md) 和答辩页映射表一起发给我做最终一致性检查。
