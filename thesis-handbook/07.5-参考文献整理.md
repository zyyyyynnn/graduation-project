# 参考文献整理

## 目标

- 在你提供原始条目信息和访问日期后，由我整理生成规范的文末参考文献列表。
- 与 [00.5-文献检索与协作工作流.md](./00.5-文献检索与协作工作流.md) 衔接，保证文献来源清楚、角色分工明确。

## 人工准备

1. 按 [00.5-文献检索与协作工作流.md](./00.5-文献检索与协作工作流.md) 收集原始文献信息。
2. 提供以下真实输入：
   - `BibTeX`（如已导出）
   - 从知网复制的中文文献条目信息
   - 从 Semantic Scholar 或其他来源整理的英文文献信息
   - 网络资源 URL 与访问日期
3. 若人工侧已经整理了文献证据映射表，也可以一并提供给协作模型，便于我核对正文使用过哪些文献、哪些条目还未入表。
4. 若人工侧已经让 GPT / Gemini 做过 deep research，可把相关条目整理建议一并提供给协作模型，但它们只能作为辅助材料，不能替代原始条目信息。

> 本文件只负责文末参考文献条目的整理与格式统一，不负责核查正文引用是否一一对应；那部分请交给 [07-全卷打磨与格式核查.md](./07-全卷打磨与格式核查.md)。

## 可直接使用的提示词

```markdown
我现在要整理文末参考文献列表。以下是我已经收集到的原始条目信息，请你直接帮我整理成规范格式。

原始条目信息：
[在此粘贴]

BibTeX（如有）：
[在此粘贴]

文献证据映射表（如有）：
[在此粘贴]

补充说明（如 deep research 报告建议、学校格式要求）：
[在此粘贴]

处理要求：
1. 只整理文末参考文献条目
2. 不新增任何未提供的文献
3. 若字段缺失，只能指出缺少哪一项，不能自行补全
4. 若学校另有格式要求，优先按学校要求处理；否则按 GB/T 7714-2015 处理

请输出：
1. 格式化后的参考文献列表
2. 缺失字段提示（如有）
```

## 使用提示

- 如果条目信息还不完整，先继续补原始来源，不要让我直接“猜”。
- 如果后续还要检查“正文引用是否与文末条目对应”，转到 [07-全卷打磨与格式核查.md](./07-全卷打磨与格式核查.md)。
- GPT / Gemini deep research 报告适合辅助你筛选文献，不适合直接替代文末条目原始信息。
- 若你想先把文献使用情况梳理清楚，可先复制 [assets-template/literature-evidence-map-template.md](./assets-template/literature-evidence-map-template.md) 建一份映射表。
- 若人工侧已经走的是“方案 A”，把 `BibTeX` 一起提供给协作模型会更稳，我可以直接读取作者、年份、期刊和 DOI 等字段。
